## Why Cease Nonsense?

<CardGrid>
	<Card title="Type Safe" icon="seti:typescript">
		Leverages TypeScript's type system for precise, context-aware linting without false positives.
	</Card>
	<Card title="React Optimized" icon="seti:react">
		Catches hook violations, component anti-patterns, and memoization mistakes in React and roblox-ts.
	</Card>
	<Card title="Performance First" icon="rocket">
		Flags performance regressions and costly patterns early, not after they hit production.
	</Card>
	<Card title="Batteries Included" icon="puzzle">
		Ships {totalRules} rules covering naming conventions, architectural patterns, and more.
	</Card>
</CardGrid>

<Aside type="note" title="What counts as type-aware?">
	These rules use parser services or the TypeScript checker in at least one code path. `naming-convention` stays
	AST-only; the rest of the rules without this badge do not need project type information.
</Aside>
