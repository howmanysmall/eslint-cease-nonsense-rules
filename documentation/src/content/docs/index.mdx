## Type-aware vs AST-only

Most rules only need the syntax tree. A smaller set uses TypeScript's checker when that is the only way to be sure.

|                                |                              Type-aware |                              AST-only |
| ------------------------------ | --------------------------------------: | ------------------------------------: |
| All rules                      |         {ruleAwareness.typeAware.count} |         {ruleAwareness.astOnly.count} |
| {ruleCategories.react.label}   |   {ruleCategories.react.typeAwareCount} |   {ruleCategories.react.astOnlyCount} |
| {ruleCategories.roblox.label}  |  {ruleCategories.roblox.typeAwareCount} |  {ruleCategories.roblox.astOnlyCount} |
| {ruleCategories.naming.label}  |  {ruleCategories.naming.typeAwareCount} |  {ruleCategories.naming.astOnlyCount} |
| {ruleCategories.general.label} | {ruleCategories.general.typeAwareCount} | {ruleCategories.general.astOnlyCount} |

<Aside type="note" title="What counts as type-aware?">
	The rule uses parser services or the TypeScript checker on at least one path. Rules without that badge stay
	AST-only. `naming-convention` is AST-only.
</Aside>
