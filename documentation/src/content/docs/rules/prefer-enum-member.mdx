Enforce using TypeScript enum members instead of raw string or number values.

## Rule Details :badge[Clean Code]{variant=success}

Enums provide a single source of truth for values. Passing raw literals or using bare keys hides intent and creates drift when enum values change. This rule requires enum member references for both `enum` and `const enum`, including object literal keys where the expected type is a mapped enum type.

<Aside type="caution" title="Avoid Drift">
	If you change an enum value but use raw strings in several places, those strings will become invalid. Using the
	member reference ensures the value is always correct.
</Aside>

## Refactoring Steps

<Steps>
1. **Locate Literals**
   Find raw strings or numbers being used in places where an Enum type is expected.

2. **Check Member Existence**
   Ensure the value you are using is actually defined in the Enum.

3. **Convert Call**
   Replace the literal with the member access (e.g., `Color.Red`).

4. **Object Keys**
   For object keys derived from enums, use computed properties: `[Color.Red]: value`.

{/* oxfmt-ignore */}

</Steps>

## Examples

<Tabs>
	<TabItem label="❌ Incorrect" icon="close">
		### Raw Values
		Bans passing raw literals where an enum is expected.

    	<Code
    		code={[
    			'enum Color { Red = "Red", Blue = "Blue" }',
    			"",
    			"// Using raw string",
    			'setColor("Red");',
    			"",
    			"// Using raw key in object",
    			"const palette = {",
    			'\tRed: "#F00"',
    			"};",
    		].join("\n")}
    		lang="tsx"
    	/>
    </TabItem>
    <TabItem label="✅ Correct" icon="check">
    	### Member References
    	Use the enum name followed by the member for all usages.

    	<Code
    		code={[
    			'enum Color { Red = "Red", Blue = "Blue" }',
    			"",
    			"// Using enum member",
    			"setColor(Color.Red);",
    			"",
    			"// Using computed key in object",
    			"const palette = {",
    			'\t[Color.Red]: "#F00"',
    			"};",
    		].join("\n")}
    		lang="tsx"
    	/>
    </TabItem>

</Tabs>

## Why Use Members?

<CardGrid stagger>
	<Card title="Global Renaming" icon="rocket">
		IDE tools can rename an enum member across your entire project safely.
	</Card>
	<Card title="Type Safety" icon="approve-check">
		Ensures you only pass values that actually exist in the enum definition.
	</Card>
	<Card title="Single Source of Truth" icon="setting">
		Change a value in one place (the enum) and it updates everywhere.
	</Card>
</CardGrid>
