Disallow JSX prop names that start with `_`.

## Rule Details :badge[Best Practice]{variant=success}

This rule bans underscore-prefixed props like `_tooltipGradient` in JSX.

<Aside type="danger" title="Props Are Public API">
	Component props are part of the component API. A leading underscore reads as "internal/private" and makes call sites
	harder to scan.
</Aside>

## Examples

<Tabs>
	<TabItem label="❌ Incorrect" icon="close">
		### Underscore-Prefixed Props

		<Code
			code={[
				"<InventoryItemTooltip",
				'\tkey="inventory-tooltip"',
				"\t_tooltipGradient={tooltipGradient}",
				"/>",
			].join("\n")}
			lang="tsx"
		/>
	</TabItem>
	<TabItem label="✅ Correct" icon="check">
		### Standard Prop Names

		<Code
			code={[
				"<InventoryItemTooltip",
				'\tkey="inventory-tooltip"',
				"\ttooltipGradient={tooltipGradient}",
				"/>",
			].join("\n")}
			lang="tsx"
		/>
	</TabItem>
</Tabs>
