Reports JSX props whose names start with `_`.

## Rule details :badge[Problem]{variant=danger}

This rule reports any JSX attribute whose name begins with `_`.

<Aside type="danger" title="Props are part of the public API">
	A leading `_` reads like an internal field name. In JSX, that usually makes the call site harder to read.
</Aside>

## Examples

<RuleExample type="fail" title="Incorrect">
	<Code
		code={["<InventoryItemTooltip", '\tkey="inventory-tooltip"', "\t_tooltipGradient={tooltipGradient}", "/>"].join(
			"\n",
		)}
		lang="tsx"
	/>
</RuleExample>

<RuleExample type="pass" title="Correct">
	<Code
		code={["<InventoryItemTooltip", '\tkey="inventory-tooltip"', "\ttooltipGradient={tooltipGradient}", "/>"].join(
			"\n",
		)}
		lang="tsx"
	/>
</RuleExample>
