Ban JSX prop names that start with `_`.

## Rule Details :badge[Best Practice]{variant=success}

This rule enforces clean, explicit prop naming by rejecting attributes like `_tooltipGradient` in JSX.

<Aside type="danger" title="Public API Should Look Public">
	Component props are part of the component API. Leading underscores imply "internal/private" and make call sites harder
	to scan.
</Aside>

## Examples

<Tabs>
	<TabItem label="❌ Incorrect" icon="close">
		### Underscore-Prefixed Props

    	<Code
    		code={[
    			"<InventoryItemTooltip",
    			'\tkey="inventory-tooltip"',
    			"\t_tooltipGradient={tooltipGradient}",
    			"/>",
    		].join("\n")}
    		lang="tsx"
    	/>
    </TabItem>
    <TabItem label="✅ Correct" icon="check">
    	### Standard Prop Names

    	<Code
    		code={[
    			"<InventoryItemTooltip",
    			'\tkey="inventory-tooltip"',
    			"\ttooltipGradient={tooltipGradient}",
    			"/>",
    		].join("\n")}
    		lang="tsx"
    	/>
    </TabItem>
</Tabs>
