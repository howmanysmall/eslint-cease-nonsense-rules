Disallow calling `Events` for the same player inside an `Events.connect()` callback.

## Rule details :badge[Problem]{variant=danger}

This rule looks for Flamework `Events` imports from the paths you configure in `eventsImportPaths`.

Inside an `Events.connect()` callback, it tracks the first callback parameter as the current player. If you call any
method on the tracked `Events` object and pass that same player as the first argument, the rule reports it.

That includes simple aliases such as `const samePlayer = player`, destructuring from objects or arrays that hold the
player, and later assignments. Calls inside nested functions are ignored.

<Aside type="caution" title="This rule needs configuration">
	If `eventsImportPaths` is empty, the rule does nothing because it does not know which `Events` import to track.
</Aside>

## Options

`eventsImportPaths` is required by the schema and defaults to an empty array.

	code={[

## Examples

<RuleExample type="fail" title="Incorrect">
	<Code
		code={[
			'import { Events } from "server/networking";',
			"",
			"Events.units.unequipUnit.connect((player, unitKey) => {",
			"\tif (unitKey.size() > 0) {",
			'\t\tEvents.promptNotification.fire(player, "error");',
			"\t}",
			"});",
			"",
			"Events.someEvent.connect((player) => {",
			"\tconst samePlayer = player;",
			"\tEvents.someEvent.fire(samePlayer, data);",
			"});",
		].join("\n")}
		lang="tsx"
	/>
</RuleExample>

<RuleExample type="pass" title="Correct">
	<Code
		code={[
			'import { Events, Functions } from "server/networking";',
			"",
			"Events.units.unequipUnit.connect((player, unitKey) => {",
			"\tif (unitKey.size() > 0) {",
			'\t\tFunctions.units.notifyPlayer(player, "error");',
			"\t}",
			"});",
			"",
			"Events.someEvent.connect((player, otherPlayer) => {",
			"\tEvents.someEvent.fire(otherPlayer, data);",
			"});",
			"",
			"function later(player: Player) {",
			"\tEvents.someEvent.fire(player, data);",
			"}",
		].join("\n")}
		lang="tsx"
	/>
</RuleExample>
