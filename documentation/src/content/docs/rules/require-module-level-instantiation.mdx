Requires configured classes to be instantiated at module scope.

## Rule details :badge[Problem]{variant=danger}

This rule does nothing until you configure a `classes` map.

Once configured, it tracks imports that match the class name and source you listed, then reports `new ClassName()` when
it happens inside a non-top-level scope. Top-level module and global scope instantiations are allowed.

The rule handles both direct class imports and member access such as `new Library.Log()` when the configured class name
matches the member name.

<Aside type="tip" title="Good fit for shared services">
	Use this for objects that should exist once per module, such as loggers, clients, or server wrappers.
</Aside>

## Examples

<RuleExample type="fail" title="Incorrect">
	<Code
		code={[
			'import Log from "@rbxts/rbxts-sleitnick-log";',
			"",
			"function runTask() {",
			"\tconst log = new Log();",
			'\tlog.Info("running");',
			"}",
		].join("\n")}
		lang="ts"
	/>
</RuleExample>

<RuleExample type="pass" title="Correct">
	<Code
		code={[
			'import Log from "@rbxts/rbxts-sleitnick-log";',
			"",
			"const log = new Log();",
			"",
			"function runTask() {",
			'\tlog.Info("running");',
			"}",
		].join("\n")}
		lang="ts"
	/>
</RuleExample>

## Options

| Option    | Type                     | Default | Notes                               |
| :-------- | :----------------------- | :------ | :---------------------------------- |
| `classes` | `Record<string, string>` | `{}`    | Maps class names to import sources. |

Example config:

```js
import ceaseNonsense from "eslint-plugin-cease-nonsense";

export default [
	{
		plugins: {
			"cease-nonsense": ceaseNonsense,
		},
		rules: {
			"cease-nonsense/require-module-level-instantiation": [
				"error",
				{
					classes: {
						Log: "@rbxts/rbxts-sleitnick-log",
						Server: "@rbxts/net",
					},
				},
			],
		},
	},
];
```
