Replace configured constructor and static method patterns with shorter forms.

## Rule details :badge[Suggestion]{variant=note}

This rule matches `new Type(...)` and `Type.method(...)` calls against the patterns you configure. When a pattern
matches, the rule reports the original call and can replace it with the configured text.

Pattern order matters. The rule tries them in the order you list them and stops at the first safe match.

<Aside type="tip" title="Why a match can be skipped">
	A pattern is skipped when the replacement would introduce a name conflict in scope, or when a captured value is too
	complex to substitute safely.
</Aside>

## Configuration

<Code
	code={[
		'import ceaseNonsense from "eslint-plugin-cease-nonsense";',
		"",
		"export default [",
		"\t{",
		'\t\tplugins: { "cease-nonsense": ceaseNonsense },',
		"\t\trules: {",
		'\t\t\t"cease-nonsense/prefer-pattern-replacements": [',
		'\t\t\t\t"error",',
		"\t\t\t\t{",
		"\t\t\t\t\tpatterns: [",
		"\t\t\t\t\t\t{",
		'\t\t\t\t\t\t\tmatch: "new Vector2($x, $x)",',
		'\t\t\t\t\t\t\treplacement: "fromUniform($x)",',
		"\t\t\t\t\t\t},",
		"\t\t\t\t\t],",
		"\t\t\t\t},",
		"\t\t\t],",
		"\t\t},",
		"\t},",
		"];",
	].join("\n")}
	lang="ts"
/>

## Options

| Option     | Type        | Default | Notes                                                 |
| :--------- | :---------- | :------ | :---------------------------------------------------- |
| `patterns` | `Pattern[]` | `[]`    | Ordered list of constructor or static method rewrites |

Each pattern object has these fields:

| Field         | Required | Description                                                                                          |
| :------------ | :------- | :--------------------------------------------------------------------------------------------------- |
| `match`       | Yes      | A constructor form like `new Vector2($x, $y)` or a static method form like `UDim2.fromScale($x, $y)` |
| `replacement` | Yes      | Replacement text such as `vec($x, $y)` or `Vector2.zero`                                             |
| `when`        | No       | Numeric conditions for captures, such as `{ x: "> 0" }`                                              |

## Pattern syntax

| Syntax                          | Meaning                                                        |
| :------------------------------ | :------------------------------------------------------------- |
| `$x`                            | Capture a value and reuse it in the replacement                |
| `_`                             | Match any single argument without capturing it                 |
| `number?`                       | Match that numeric literal, `undefined`, or a missing argument |
| repeated capture, like `$x, $x` | Both arguments must be the same value                          |

`when` conditions only work for captures that resolve to numeric constants.

## Examples

<Code
	code={[
		"pattern({",
		'\tmatch: "new Vector2($x, $x)",',
		'\treplacement: "fromUniform($x)",',
		"});",
		"",
		"pattern({",
		'\tmatch: "UDim2.fromScale($x, $x)",',
		'\treplacement: "scale($x)",',
		'\twhen: { x: "> 0" },',
		"});",
		"",
		"pattern({",
		'\tmatch: "new Vector2($x, 0?)",',
		'\treplacement: "fromX($x)",',
		"});",
	].join("\n")}
	lang="ts"
/>
