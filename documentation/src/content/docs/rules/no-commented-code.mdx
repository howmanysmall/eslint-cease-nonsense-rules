Reports comments that look like real code.

## Rule details :badge[Suggestion]{variant=note}

The rule groups adjacent line comments, checks block comments on their own, and tries to parse the comment text as
JavaScript or TypeScript. If the result looks like real code, it reports the whole comment range and offers a removal
suggestion.

It intentionally skips a lot of comments that would be noisy to flag, including empty comments, JSDoc, linter
directives, many plain-English notes, URLs, and a few short parseable forms that are common in prose.

<Aside type="tip" title="What to do instead">
	Delete dead code. If the comment was carrying context, keep that context as a short explanation or move it to an
	issue.
</Aside>

## Examples

<RuleExample type="fail" title="Incorrect">
	<Code
		code={[
			"function calculate(x: number) {",
			"\t// const result = x * 2;",
			"\t// return result;",
			"\treturn x + 1;",
			"}",
		].join("\n")}
		lang="ts"
	/>
</RuleExample>

<RuleExample type="pass" title="Correct">
	<Code
		code={[
			"function calculate(x: number) {",
			"\t// TODO: switch back to multiplication after the API lands",
			"\t// The offset is temporary because the upstream value can overflow.",
			"\treturn x + 1;",
			"}",
		].join("\n")}
		lang="ts"
	/>
</RuleExample>

## What it catches

- adjacent `//` lines that form a code block
- `/* ... */` comments that parse as statements or expressions
- incomplete commented code where the rule can infer missing braces and still parse it
- JSX-like commented code when the parser can recover with TSX parsing

The rule does not auto-fix by default. It adds a suggestion to remove the commented block.
