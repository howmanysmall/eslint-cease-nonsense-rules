Replaces direct provider chains with a local `ContextStack` component when one is available.

## Rule details :badge[Suggestion]{variant=note}

This rule looks for JSX like `A.Provider -> B.Provider -> children` and reports it when your project already has a local
`ContextStack` component.

It stays conservative:

- It only reports chains with more than one provider.
- It skips nested providers that are already part of a larger chain.
- It skips wrappers that also render siblings or extra content.
- It only auto-fixes when the current file already imports exactly one local `ContextStack` identifier and the file is
  `.jsx` or `.tsx`.

<Aside type="tip" title="Discovery is local">
	The rule does not assume a package import. It only works with a local `ContextStack` component that it can discover
	in your project or inspect through a relative import.
</Aside>

## Examples

<RuleExample type="fail" title="Incorrect">
	<Code
		code={[
			"return (",
			"\t<ThemeContext.Provider value={theme}>",
			"\t\t<LocaleContext.Provider value={locale}>",
			"\t\t\t<App />",
			"\t\t</LocaleContext.Provider>",
			"\t</ThemeContext.Provider>",
			");",
		].join("\n")}
		lang="tsx"
	/>
</RuleExample>

<RuleExample type="pass" title="Correct">
	<Code
		code={[
			"return (",
			"\t<ContextStack",
			"\t\tproviders={[",
			"\t\t\t<ThemeContext.Provider value={theme} />,",
			"\t\t\t<LocaleContext.Provider value={locale} />,",
			"\t\t]}",
			"\t>",
			"\t\t<App />",
			"\t</ContextStack>",
			");",
		].join("\n")}
		lang="tsx"
	/>
</RuleExample>
