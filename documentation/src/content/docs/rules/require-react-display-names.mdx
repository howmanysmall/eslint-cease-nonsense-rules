Require `displayName` on exported `memo(...)` and `createContext(...)` values.

## Rule Details :badge[Best Practice]{variant=success}

Components wrapped in `memo` and contexts from `createContext` can lose useful names in React DevTools unless you set
`displayName`. This rule makes you keep that name.

<Aside type="tip" title="Production Debugging">
	Minification often strips function names. `displayName` is a string literal that survives, so you can still read
	DevTools and stack traces.
</Aside>

## Refactoring Path

<Steps>
1. **Locate Exports**
   Find `export default memo(...)` or `export const MyContext = createContext(...)` calls.

2. **Assign to Constant** Instead of exporting the result directly, assign it to a named variable (e.g.,
   `const MyComponent = memo(...)`).

3. **Set Display Name** Append a `.displayName` assignment: `MyComponent.displayName = "MyComponent"`.

4. **Export Named** Export the named variable. You can still use `export default` for the named variable afterwards.

{/* oxfmt-ignore */}

</Steps>

## Examples

<Tabs>
	<TabItem label="❌ Incorrect" icon="close">
		### Anonymous Exports
		Bans direct exports that hide the component name from the debugger.

    	<Code
    		code={[
    			"// Direct memo export",
    			"export default memo(MyComponent);",
    			"",
    			"// Missing displayName",
    			"export const MyContext = React.createContext(undefined);",
    		].join("\n")}
    		lang="tsx"
    	/>
    </TabItem>
    <TabItem label="✅ Correct" icon="check">
    	### Labeled Components
		Name your exports so DevTools and stack traces stay readable.

    	<Code
    		code={[
    			'const MyComponent = memo(MyComponentNoMemo);',
    			'MyComponent.displayName = "MyComponent";',
    			"export default MyComponent;",
    			"",
    			"const MyContext = React.createContext(undefined);",
    			'MyContext.displayName = "MyContext";',
    			"export default MyContext;",
    		].join("\n")}
    		lang="tsx"
    	/>
    </TabItem>

</Tabs>

## Why is this important?

<CardGrid stagger>
	<Card title="DevTools Mapping" icon="list-format">
		Ensures your component tree doesn't just show `<Memo(Anonymous)>` everywhere.
	</Card>
	<Card title="Profiling" icon="bolt">
		Helps the React Profiler attribute performance bottlenecks to specific named components.
	</Card>
	<Card title="Refactoring Safety" icon="setting">
		Assigning to a local constant before exporting makes the code easier to grep and refactor.
	</Card>
</CardGrid>
