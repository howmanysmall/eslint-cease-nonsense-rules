Require `displayName` property on exported `React.memo` components and `React.createContext` contexts for better debugging.

## Rule Details

React components wrapped in `memo` and contexts created with `createContext` lose their names in React DevTools unless you explicitly set `displayName`. This rule enforces proper naming patterns to maintain debuggability.

## Examples

### ❌ Incorrect

**Direct memo export (worst pattern)**

<Code
	code={[
		'import { memo } from "@rbxts/react";',
		"",
		"function ButtonNoMemo() {",
		"\treturn <button />;",
		"}",
		"",
		"// Error: Do not export memo result directly",
		"export default memo(ButtonNoMemo);",
	].join("\n")}
	lang="typescript"
	key="import-memo-from-rbx-3c3uog"
/>

**Missing displayName on memo**

<Code
	code={[
		'import { memo } from "@rbxts/react";',
		"",
		"function ChecklistClaimButtonNoMemo() {",
		"\treturn <frame />;",
		"}",
		"",
		"// Error: Missing displayName",
		"const ChecklistClaimButton = memo(ChecklistClaimButtonNoMemo);",
		"export default ChecklistClaimButton;",
	].join("\n")}
	lang="typescript"
	key="import-memo-from-rbx-2r8d0m"
/>

**Direct createContext export**

<Code
	code={[
		'import React from "@rbxts/react";',
		"",
		"// Error: Do not export createContext result directly",
		"export default React.createContext<string | undefined>(undefined);",

	lang="typescript"
	key="import-react-from-rb-24szhs"
/>

**Missing displayName on context**

<Code
	code={[
		'import React from "@rbxts/react";',
		"",
		"// Error: Missing displayName",
		"const ErrorBoundaryContext = React.createContext<unknown>(undefined);",

	].join("\n")}
	lang="typescript"

### ✅ Correct

**Proper memo pattern**

<Code
	code={[
		'import { memo } from "@rbxts/react";',
		"",
		"function ChecklistClaimButtonNoMemo() {",
		"\treturn <frame />;",
		"}",
		"",
		"export const ChecklistClaimButton = memo(ChecklistClaimButtonNoMemo);",
		'ChecklistClaimButton.displayName = "ChecklistClaimButton";',
		"export default ChecklistClaimButton;",
	].join("\n")}
	lang="typescript"
	key="import-memo-from-rbx-2atih9"
/>

**Proper context pattern**

<Code
	code={[
		'import React from "@rbxts/react";',
		"",
		"const ErrorBoundaryContext = React.createContext<unknown>(undefined);",
		'ErrorBoundaryContext.displayName = "ErrorBoundaryContext";',
		"export default ErrorBoundaryContext;",
	].join("\n")}
	lang="typescript"
	key="import-react-from-rb-2g2arc"
/>

**Internal (non-exported) usage is fine**

<Code
	code={[
		'import { memo, createContext } from "@rbxts/react";',
		"",
		"// Not exported, so no displayName required",
		"const InternalContext = createContext<number>(0);",
		"",
		"function InnerComponentNoMemo() {",
		"\treturn <div />;",
		"}",
		"",
		"// Not exported, so no displayName required",
		"const InnerComponent = memo(InnerComponentNoMemo);",
		"",
		"// Only this is exported",
		"export function Container() {",
		"\treturn <InnerComponent />;",
		"}",
	].join("\n")}
	lang="typescript"

## Options

### `environment`

- Type: `"roblox-ts" | "standard"`
- Default: `"roblox-ts"`

Controls which React package sources are checked.

- `"roblox-ts"`: Checks imports from `@rbxts/react` and `@rbxts/roact`
- `"standard"`: Checks imports from `react` and `react-dom`

**Configuration for standard React**

<Code
	code={[
		"{",
		'  "cease-nonsense/require-react-display-names": ["error", {',
		'    "environment": "standard"',
		"  }]",
		"}",
	].join("\n")}
	lang="typescript"
	key="block-7-1qcetu"
/>

## Why Is This Bad?

### DevTools Debugging

Without `displayName`, memoized components and contexts appear as anonymous or with unhelpful names in React DevTools:

<Code
	code={[
		"<Memo(Anonymous)>     // vs <ChecklistClaimButton>",
		"<Context.Provider>    // vs <ErrorBoundaryContext.Provider>",
	].join("\n")}
	lang="text"
	key="memo-anonymous-vs-ch-2cfybb"
/>

### Production Debugging

Minification often strips function names. `displayName` survives minification and appears in error stack traces and profiler data.

### Direct Export Problems

Exporting `memo(Component)` directly:

1. Loses the component name completely
2. Makes it impossible to set `displayName` after the fact
3. Prevents named exports alongside default

## When Not To Use It

- For non-exported memo/context usage (the rule already ignores these)
- If you're using a bundler/transformer that automatically adds displayName
- For very simple internal components where debugging isn't a concern

## Migration Guide

### Before

<Code
	code={[
		'import { memo } from "@rbxts/react";',
		"",
		"function ComponentNoMemo(props: Props) {",
		"\treturn <frame />;",
		"}",
		"",
		"export default memo(ComponentNoMemo);",
	].join("\n")}
	lang="typescript"
	key="import-memo-from-rbx-39p6vo"
/>

### After

<Code
	code={[
		'import { memo } from "@rbxts/react";',
		"",
		"function ComponentNoMemo(props: Props) {",
		"\treturn <frame />;",
		"}",
		"",
		"export const Component = memo(ComponentNoMemo);",
		'Component.displayName = "Component";',
		"export default Component;",
	].join("\n")}
	lang="typescript"
	key="import-memo-from-rbx-1iotcq"
/>
