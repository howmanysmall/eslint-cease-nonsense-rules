Require React function component props to be read-only.

## Rule details :badge[Suggestion]{variant=note}

This rule checks React function components and reports prop types that are still mutable.

With type information enabled, it can validate named prop types, `React.FC`, `memo(...)`, and `forwardRef(...)`
patterns. Without type information, it still auto-fixes inline type literals on component parameters.

<Aside type="tip" title="What auto-fix covers">
	Auto-fix inserts `readonly` for inline prop type literals such as `props: { value: string }`. Named interfaces and
	type aliases may still report, but they are not rewritten by this rule.
</Aside>

## Examples

<RuleExample type="fail" title="Incorrect">
		### Mutable props

        	<Code
        		code={[
        			"function Button(props: { label: string; onClick: () => void }) {",
        			"\treturn <textbutton Text={props.label} />;",
        			"}",
        		].join("\n")}
        		lang="tsx"
        	/>
    </RuleExample>

<RuleExample type="pass" title="Correct">
		### Read-only props

        	<Code
        		code={[
        			"type ButtonProps = {",
        			"\treadonly label: string;",
        			"\treadonly onClick: () => void;",
        			"};",
        			"",
        			"function Button(props: ButtonProps) {",
        			"\treturn <textbutton Text={props.label} />;",
        			"}",
        		].join("\n")}
        		lang="tsx"
        	/>
    </RuleExample>

## Notes

The rule is aimed at component names that look like React components, so helper functions such as `makeConfig` are not
reported.

Destructured props are not auto-fixed. If you want full coverage for shared prop types, run ESLint with type-aware
parsing.
