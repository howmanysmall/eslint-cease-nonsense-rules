Require `readonly` on prop types.

## Rule Details :badge[Safety]{variant=success}

Props are inputs. Mutating them is a bug waiting to happen. This rule requires prop types to use `readonly` so
accidental mutation fails at compile time.

<Aside type="tip" title="Immutability">
	`readonly` makes it harder to accidentally "fix" a value by mutating it inside a component.
</Aside>

## Refactoring Steps

<Steps>
1. **Identify Prop Types**
   Find `interface` or `type` definitions used for React component props.

2. **Add Readonly** Prefix every property in the interface with the `readonly` keyword.

3. **Check Usage** Ensure your component code does not attempt to assign to these props (e.g.,
   `props.name = "New Name"`).

4. **Verify Interfaces** If you extend other interfaces, ensure those are also immutable or use `Readonly<T>` for the
   intersection.

    {/* oxfmt-ignore */}

</Steps>

## Examples

<Tabs>
	<TabItem label="❌ Incorrect" icon="close">
		### Mutable Props
		Bans prop definitions that allow re-assignment.

    	<Code
    		code={[
    			"interface ButtonProps {",
    			"\tlabel: string;",
    			"\tonClick: () => void;",
    			"}",
    			"",
    			"function Button({ label }: ButtonProps) {",
    			"\treturn <textbutton Text={label} />;",
    			"}",
    		].join("\n")}
    		lang="tsx"
    	/>
    </TabItem>
    <TabItem label="✅ Correct" icon="check">
    	### Read-Only Props
    	Use the `readonly` modifier for every property in your interface.

    	<Code
    		code={[
    			"interface ButtonProps {",
    			"\treadonly label: string;",
    			"\treadonly onClick: () => void;",
    			"}",
    			"",
    			"function Button({ label }: ButtonProps) {",
    			"\treturn <textbutton Text={label} />;",
    			"}",
    		].join("\n")}
    		lang="tsx"
    	/>
    </TabItem>

</Tabs>

## Why Enforce This?

<CardGrid stagger>
	<Card title="Bug Prevention" icon="bolt">
		Prevents developers from accidentally overwriting props within the component body.
	</Card>
	<Card title="Documentation" icon="list-format">
		Clearly signals to other developers that these values are strictly inputs and should not be modified.
	</Card>
	<Card title="Performance" icon="rocket">
		Immutability makes it safer to use optimizations like `React.memo` as you can trust prop references.
	</Card>
</CardGrid>
