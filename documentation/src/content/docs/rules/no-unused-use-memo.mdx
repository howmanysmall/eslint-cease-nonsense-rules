Reports `useMemo` calls whose return value is ignored.

## Rule details :badge[Problem]{variant=danger}

This rule reports `useMemo(...)` when it is used as a standalone statement, including `void useMemo(...)`. If you are
running side effects, use `useEffect`. If you need memoization, keep and use the returned value.

By default, the rule looks for React imports in the `roblox-ts` environment. Set `environment` to `"standard"` to use
`react` instead.

<Aside type="danger" title="useMemo is not an effect hook">
	A discarded memo value usually means the hook is being used for side effects or to dodge effect rules.
</Aside>

## Options

| Option        | Type         | Default     | What it does  |
| ------------- | ------------ | ----------- | ------------- | -------------------------------------------- |
| `environment` | `"roblox-ts" | "standard"` | `"roblox-ts"` | Chooses which React package imports to track |

## Examples

<RuleExample type="fail" title="Incorrect">
	<Code
		code={[
			'import { useMemo } from "react";',
			"",
			"useMemo(() => {",
			"\ttrackAnalytics();",
			"}, [eventName]);",
			"",
			"void useMemo(() => buildConfig(eventName), [eventName]);",
		].join("\n")}
		lang="tsx"
	/>
</RuleExample>

<RuleExample type="pass" title="Correct">
	<Code
		code={[
			'import { useEffect, useMemo } from "react";',
			"",
			"useEffect(() => {",
			"\ttrackAnalytics();",
			"}, [eventName]);",
			"",
			"const config = useMemo(() => buildConfig(eventName), [eventName]);",
		].join("\n")}
		lang="tsx"
	/>
</RuleExample>
