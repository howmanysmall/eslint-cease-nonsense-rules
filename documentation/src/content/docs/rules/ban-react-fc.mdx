Bans `React.FC` and similar component type annotations.

## Rule Details :badge[Best Practice]{variant=success}

`React.FC` and similar type annotations break debug information in React DevTools, hide component display names, and
encourage poor patterns like implicit children typing. Use explicit function declarations with typed props instead to
ensure maximum type safety and a clean debugging experience.

<Aside type="danger" title="Why Ban React.FC?">
	`React.FC` makes your components harder to debug and less type-safe. It encourages using arrow functions which often
	show up as `Anonymous` in DevTools.
</Aside>

## Refactoring Steps

<Steps>
1. **Identify React.FC**
   Look for variable declarations using `: React.FC<Props>` or `: FC<Props>`.
2. **Convert to Function** Change `const Component: React.FC<Props> = (...)` to `function Component(...)`.
3. **Type Props Directly** Move the prop type from the variable annotation to the function parameter list:
   `(props: Props)`.
4. **Clean Up** Check if your component uses `children`. If so, explicitly add `children: ReactNode` to your props
   interface.
{/* oxfmt-ignore */}

</Steps>

## Examples

<Tabs>
	<TabItem label="❌ Incorrect" icon="close">
		### Type Annotations
		Bans the use of generic wrapper types for component variables.

    	<Code
    		code={[
    			"export const MyComponent: React.FC<Props> = ({ children }) => {",
    			"\treturn <div>{children}</div>;",
    			"};",
    			"",
    			"const Button: FC<ButtonProps> = ({ label }) => <button>{label}</button>;",
    		].join("\n")}
    		lang="tsx"
    	/>
    </TabItem>
    <TabItem label="✅ Correct" icon="check">
    	### Function Declarations
    	Use standard TypeScript function signatures for your components.

    	<Code
    		code={[
    			"export function MyComponent({ children }: Props) {",
    			"\treturn <div>{children}</div>;",
    			"}",
    			"",
    			"function Button({ label }: ButtonProps) {",
    			"\treturn <button>{label}</button>;",
    			"}",
    		].join("\n")}
    		lang="tsx"
    	/>
    </TabItem>

</Tabs>

## Problems with React.FC

<CardGrid stagger>
	<Card title="DevTools Issues" icon="setting">
		Obscures names in React DevTools, making it difficult to navigate your component tree.
	</Card>
	<Card title="Implicit Children" icon="warning">
		Automatically includes `children`, which can lead to bugs if a component shouldn't accept them.
	</Card>
	<Card title="Generic Support" icon="bolt">
		`React.FC` is difficult to use with generic components, whereas function declarations handle them naturally.
	</Card>
</CardGrid>
