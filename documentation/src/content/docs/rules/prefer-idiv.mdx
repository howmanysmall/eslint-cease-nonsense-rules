Prefer `.idiv()` for integer division instead of `math.floor(x / y)`.

## Rule Details :badge[Roblox]{variant=note}

In `roblox-ts`, the `.idiv()` method compiles to Luau's `//` (floor division) operator. Using `math.floor(x / y)` is
less idiomatic and slightly less efficient than using the dedicated integer division operator.

## Examples

<RuleExample type="fail" title="Incorrect">
		### Using math.floor for Division
		Avoid wrapping division in `math.floor()` when you want integer division.

    	<Code
    		code={[
    			"const quotient = math.floor(x / y);",
    			"const result = math.floor((a + b) / c);",
    			"const value = math.floor(100 / 3);",
    		].join("\n")}
    		lang="tsx"
    	/>
    </RuleExample>

<RuleExample type="pass" title="Correct">
    	### Using .idiv() Method
    	Use the `.idiv()` method for integer division.

    	<Code
    			code={[
    				"const quotient = x.idiv(y);",
    				"const result = (a + b).idiv(c);",
    				"const value = (100).idiv(3);",
    				"const nested = (a / b).idiv(c);",
    			].join("\n")}
    		lang="tsx"
    	/>
    </RuleExample>

## Why use .idiv()?

<CardGrid stagger>
	<Card title="Idiomatic" icon="code">
		`.idiv()` is the roblox-ts convention for integer division, compiling directly to Luau's `//` operator.
	</Card>
	<Card title="Performance" icon="bolt">
		Slightly more efficient than floating-point division followed by `math.floor()`.
	</Card>
	<Card title="Readability" icon="book">
		Intent is clearer: you want integer division, not a floor operation on a float result.
	</Card>
	<Card title="Auto-fixable" icon="wrench">
		Matches are automatically fixed by ESLint's `--fix` command.
	</Card>
</CardGrid>
