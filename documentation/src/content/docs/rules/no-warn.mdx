Reports direct calls to the global `warn()` function.

## Rule details :badge[Problem]{variant=danger}

This rule reports `warn(...)` when it is called as a global function. The rule does not take options.

The message points you toward a logging package instead, because raw warning output is hard to filter and harder to work
with once a codebase gets larger.

<Aside type="tip" title="This rule only matches the global function">
	It reports `warn(...)`. It does not report method calls like `logger.warn(...)`.
</Aside>

## Examples

<RuleExample type="fail" title="Incorrect">
	<Code
		code={[
			'warn("API endpoint deprecated");',
			'warn("Missing config property:", key);',
			'warn("Unexpected state in component");',
		].join("\n")}
		lang="tsx"
	/>
</RuleExample>

<RuleExample type="pass" title="Correct">
	<Code
		code={[
			'log.Warning("API endpoint deprecated");',
			'log.Warning("Missing config property:", key);',
			'logger.warn("Unexpected state in component");',
		].join("\n")}
		lang="tsx"
	/>
</RuleExample>
