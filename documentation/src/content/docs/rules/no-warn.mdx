Bans use of `warn()` function calls. Use `log.Warning` instead for structured logging.

## Rule Details

The `warn()` function is a low-level warning mechanism that lacks structure, context, and log level control. Using a proper logging system like [`Log`](https://sleitnick.github.io/RbxUtil/api/Log) provides better debugging capabilities and production logging.

<Aside type="tip" title="Consistent logging">
	Using `log.Warning` instead of `warn()` ensures all your logs go through a unified logging system with consistent
	formatting and level control.
</Aside>

## Examples

### ❌ Incorrect

<Code
	code={[
		'warn("Warning message");',
		'warn("Deprecated function called");',
		'warn("Invalid configuration:", config);',
		"",
		"function validateInput(input: string) {",
		"\tif (input.length === 0) {",
		'\t\twarn("Empty input detected");',
		"\t}",
		"}",
	].join("\n")}
	lang="typescript"
	title="incorrect.ts"
/>

### ✅ Correct

<Code
	code={[
		'log.Warning("Warning message");',
		'log.Warning("Deprecated function called");',
		'log.Warning("Invalid configuration:", config);',
		"",
		"function validateInput(input: string) {",
		"\tif (input.length === 0) {",
		'\t\tlog.Warning("Empty input detected");',
		"\t}",
		"}",
	].join("\n")}
	lang="typescript"
	title="correct.ts"
/>

## Why Use Log Instead?

### Structured Logging

`Log` provides structured output with timestamps, log levels, and context:

<Code
	code={[
		"// warn() - unstructured, no context",
		'warn("API endpoint deprecated");',
		"",
		"// log.Warning - structured with level and automatic timestamp",
		'log.Warning("API endpoint deprecated");',
		"// Output: [WARNING] 2025-12-31 12:34:56 - API endpoint deprecated",
	].join("\n")}
	lang="typescript"
	title="structured-logging.ts"
/>

### Better Context

Add metadata and context to warnings:

<Code
	code={[
		'log.Warning("Deprecated API used", {',
		'\tendpoint: "/api/v1/users",',
		'\treplacement: "/api/v2/users",',
		'\tdeprecationDate: "2025-01-01",',
		"});",
	].join("\n")}
	lang="typescript"
	title="context-logging.ts"
/>

### Log Levels

The warning level sits between informational and error messages:

<Code
	code={[
		'log.Debug("Detailed debug info"); // Development only',
		'log.Info("User action completed"); // General information',
		'log.Warning("Deprecated API used"); // ⚠️ Warnings',
		'log.Error("Operation failed"); // Errors',
	].join("\n")}
	lang="typescript"
	title="log-levels.ts"
/>

## When Not To Use It

<Aside type="caution">
	If you're writing quick scripts or prototypes where structured logging isn't necessary, you may disable this rule.
	However, it's recommended to use proper logging even in development to build good habits.
</Aside>
