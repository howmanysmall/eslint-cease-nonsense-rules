Disallow empty array literals. Use constructor form for empty arrays instead.

## Rule Details :badge[TypeScript]{variant=note}

This rule reports `[]` when it has zero elements.

- If the array element type is explicit, the fixer uses `new Array<T>()`.
- If `inferTypeForEmptyArrayFix` is enabled, it can infer `T` from contextual typing.
- If no type is available, the rule reports and offers `new Array()` as a suggestion.
- Non-empty literals like `[1, 2, 3]` are allowed.

### Context Configuration

You can disable individual allowed contexts by setting `allowedEmptyArrayContexts` values to `false`. This lets you tune
which safe usages keep `[]` (for example call arguments for hook dependencies, `for..of` defaults, JSX props, and so
on).

<Aside type="tip" title="Works Well With array-type-generic">
	Use this rule together with `array-type-generic` if you want consistent type/value array conventions.
</Aside>

## Examples

<RuleExample type="fail" title="Incorrect">
	<Code
		code={[
			"const values: Array<string> = [];",
			"function readAll(input: Array<number> = []) {",
			"\treturn input;",
			"}",
		].join("\n")}
		lang="tsx"
	/>
</RuleExample>

<RuleExample type="pass" title="Correct">
	<Code
		code={[
			"const values: Array<string> = new Array<string>();",
			"function readAll(input: Array<number> = new Array<number>()) {",
			"\treturn input;",
			"}",
			"const seeded = [1, 2, 3];",
		].join("\n")}
		lang="tsx"
	/>
</RuleExample>
