Report instance methods that never use `this` or `super`.

## Rule details :badge[Problem]{variant=danger}

This rule checks non-static class methods with `kind: "method"`. Getters, setters, constructors, and static methods are
ignored.

By default it checks public, protected, and private methods. If a method body never touches `this` or `super`, the rule
reports it.

The scan walks the whole method body, including nested functions. If a nested callback uses `this`, the method is
treated as using `this` too.

<Aside type="tip" title="Why this exists">
	The rule is aimed at `roblox-ts`, where extra instance methods add metatable overhead. If a method is really a
	helper, keeping it at module scope is often a better fit.
</Aside>

## Options

All three options default to `true`.

- `checkPublic`
- `checkProtected`
- `checkPrivate`

## Examples

<RuleExample type="fail" title="Incorrect">
	<Code
		code={["class MathSystem {", "\tprivate calculate(x: number): number {", "\t\treturn x * x;", "\t}", "}"].join(
			"\n",
		)}
		lang="tsx"
	/>
</RuleExample>

<RuleExample type="pass" title="Correct">
	<Code
		code={[
			"function calculate(x: number): number {",
			"\treturn x * x;",
			"}",
			"",
			"class MathSystem {",
			"\tpublic run(value: number): number {",
			"\t\treturn calculate(value);",
			"\t}",
			"",
			"\tprotected log(): void {",
			"\t\tprint(this);",
			"\t}",
			"}",
		].join("\n")}
		lang="tsx"
	/>
</RuleExample>
