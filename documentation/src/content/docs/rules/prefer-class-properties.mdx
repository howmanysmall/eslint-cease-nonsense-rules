Moves simple literal defaults out of the constructor and into class fields.

## Rule details :badge[Suggestion]{variant=note}

By default, this rule reports `this.property = value` assignments inside constructors when `value` is known at class
definition time. That includes primitive literals, literal arrays, and object literals without spreads or computed
properties.

The rule only checks constructor assignments to `this` properties. If the value depends on constructor parameters or
runtime work, it is left alone.

<Aside type="tip" title="There is also a reverse mode">
	Set the rule to `"never"` if you want to ban instance field declarations and keep initialization in the constructor
	instead.
</Aside>

## Examples

<RuleExample type="fail" title="Incorrect">
	<Code
		code={[
			"class User {",
			"\tconstructor() {",
			'\t\tthis.name = "Unknown";',
			"\t\tthis.age = 0;",
			"\t\tthis.flags = [true, false];",
			"\t}",
			"}",
		].join("\n")}
		lang="ts"
	/>
</RuleExample>

<RuleExample type="pass" title="Correct">
	<Code
		code={[
			"class User {",
			'\tname = "Unknown";',
			"\tage = 0;",
			"\tflags = [true, false];",
			"}",
			"",
			"class Service {",
			"\tconstructor(options: Options) {",
			"\t\tthis.options = options;",
			"\t}",
			"}",
		].join("\n")}
		lang="ts"
	/>
</RuleExample>

## Options

| Option     | Default | What it does                                                                           |
| ---------- | ------- | -------------------------------------------------------------------------------------- |
| `"always"` | yes     | Reports constructor assignments of simple literal values and prefers class fields.     |
| `"never"`  | no      | Reports non-static class property declarations and prefers constructor initialization. |

```js
// eslint.config.js
import ceaseNonsense from "eslint-plugin-cease-nonsense";

export default [
	{
		plugins: {
			"cease-nonsense": ceaseNonsense,
		},
		rules: {
			"cease-nonsense/prefer-class-properties": ["error", "never"],
		},
	},
];
```
