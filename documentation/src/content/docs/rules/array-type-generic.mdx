Use one array type style everywhere: `Array<T>` / `ReadonlyArray<T>`.

## Rule Details :badge[TypeScript]{variant=note}

This rule reports bracket array syntax and fixes it to generic syntax.

1. `T[]` -> `Array<T>`
2. `readonly T[]` -> `ReadonlyArray<T>`
3. Nested arrays (`string[][]`) -> nested generic forms (`Array<Array<string>>`)
4. Tuple arrays (`[A, B][]`) -> `Array<[A, B]>`

Standalone tuples remain valid.

<Aside type="tip" title="Tuple Types Stay Intact">
	`[x: number, y: number]` is valid and stays unchanged. Only array wrappers are rewritten.
</Aside>

## Examples

<RuleExample type="fail" title="Incorrect">
	<Code
		code={[
			"type Names = string[];",
			"type Values = readonly number[];",
			"type Pairs = [number, string][];",
			"type Grid = string[][];",
		].join("\n")}
		lang="tsx"
	/>
</RuleExample>

<RuleExample type="pass" title="Correct">
	<Code
		code={[
			"type Names = Array<string>;",
			"type Values = ReadonlyArray<number>;",
			"type Pairs = Array<[number, string]>;",
			"type Grid = Array<Array<string>>;",
			"type Point = [x: number, y: number];",
		].join("\n")}
		lang="tsx"
	/>
</RuleExample>

## Why use this?

<CardGrid stagger>
	<Card title="Single Style" icon="list-format">
		Review is faster when the codebase only has one array type format.
	</Card>
	<Card title="Clear Nesting" icon="magnifier">
		Nested arrays and readonly arrays are easier to read in generic form.
	</Card>
	<Card title="Safe Autofix" icon="bolt">
		The fixer rewrites the full type chain and keeps tuple semantics intact.
	</Card>
</CardGrid>
