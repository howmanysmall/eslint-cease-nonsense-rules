Require `SCREAMING_SNAKE_CASE` variables to be `const` declarations at module scope.

## Rule details :badge[Suggestion]{variant=note}

This rule checks simple identifier names that match `SCREAMING_SNAKE_CASE`. Those bindings must use `const`, and they
must live at the top level of the module.

If a value needs to change, or it only makes sense inside a function or block, use a different name.

<Aside type="tip" title="What it ignores">
	Destructuring is not checked by this rule. `const {FOO} = config` and `const [FOO] = values` are both allowed.
</Aside>

## Examples

<RuleExample type="fail" title="Incorrect">
		### Local or mutable uppercase names

        	<Code
        		code={[
        			"function calculate() {",
        			"\tconst PI = 3.14;",
        			"}",
        			"",
        			"let MAX_RETRIES = 3;",
        		].join("\n")}
        		lang="ts"
        	/>
    </RuleExample>

<RuleExample type="pass" title="Correct">
		### Module-scope constants

        	<Code
        		code={[
        			"const PI = 3.14;",
        			"const MAX_RETRIES = 3;",
        			"",
        			"function calculate(radius: number) {",
        			"\treturn radius * radius * PI;",
        			"}",
        		].join("\n")}
        		lang="ts"
        	/>
    </RuleExample>

## Notes

In `script` files, the rule also allows a first-level function scope. That matches common CommonJS wrapper patterns.
