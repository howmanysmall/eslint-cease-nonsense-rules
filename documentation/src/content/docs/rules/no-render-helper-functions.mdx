Reports ordinary functions that return JSX where a component would be a better fit.

## Rule details :badge[Suggestion]{variant=note}

The rule checks function declarations and functions assigned to variables. If a non-PascalCase function returns JSX or a
fragment, or is annotated as `ReactNode` or `ReactElement`, it is reported.

PascalCase component names are allowed. Hook names that start with `use` are allowed. Inline callbacks like
`items.map((item) => <Item key={item.id} />)` are allowed. Functions nested inside a PascalCase component are also
ignored.

<Aside type="tip" title="What the rule does not check">
	It does not inspect class methods, and it does not report helper functions that return plain values like strings or
	objects.
</Aside>

## Examples

<RuleExample type="fail" title="Incorrect">
		### Helper function returning JSX

        	<Code
        		code={[
        			"function createLabel(text: string) {",
        			"\treturn <TextLabel text={text} />;",
        			"}",
        			"",
        			"const renderItem = (item: string) => <div>{item}</div>;",
        		].join("\n")}
        		lang="tsx"
        	/>
    </RuleExample>

<RuleExample type="pass" title="Correct">
	    	### Component and inline callback

        	<Code
        		code={[
        			"function Label(props: { text: string }) {",
        			"\treturn <TextLabel text={props.text} />;",
        			"}",
        			"",
        			"items.map((item) => <Label key={item} text={item} />);",
        		].join("\n")}
        		lang="tsx"
        	/>
    </RuleExample>
