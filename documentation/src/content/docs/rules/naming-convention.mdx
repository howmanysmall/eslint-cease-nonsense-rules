Enforce naming conventions for TypeScript constructs.

## Rule Details :badge[Performance]{variant=success}

This rule enforces naming conventions for interfaces, types, and other TypeScript constructs. Unlike the standard
`typescript-eslint` rule, this one is optimized for extreme speed by using AST-only analysis. This makes it ideal for
large codebases where full type-checking during linting is too slow.

<Aside type="note" title="Design Philosophy">
	We believe naming conventions should be fast and simple. By avoiding full type checking, this rule provides nearly
	instantaneous feedback as you type.
</Aside>

## Configuration Workflow

<Steps>
1. **Identify Selectors**
   Choose which constructs to check (e.g., `interface`, `typeAlias`, `enum`).
2. **Select Formats** Define allowed casing formats (e.g., `PascalCase`, `camelCase`, `UPPER_CASE`).
3. **Define Custom Rules** Use regex to add specific constraints, like banning the `I` prefix on interfaces.
4. **Verify Performance** Ensure your lint times remain low; this rule handles thousands of nodes in milliseconds.
{/* oxfmt-ignore */}

</Steps>

## Selectors & Formats

| Selector    | Common Format | Description            |
| :---------- | :------------ | :--------------------- |
| `interface` | `PascalCase`  | Standard TS interfaces |
| `typeAlias` | `PascalCase`  | Type definitions       |
| `variable`  | `camelCase`   | Standard variables     |
| `constant`  | `UPPER_CASE`  | Global constants       |

## Why use this over TS-ESLint?
