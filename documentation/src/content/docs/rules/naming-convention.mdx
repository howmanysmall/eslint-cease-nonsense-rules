Fast naming convention checks for TypeScript code.

## Rule Details :badge[Performance]{variant=success}

Checks casing and naming for interfaces, type aliases, enums, variables, and more. It stays AST-only (no type graph), so
it runs quickly even on large projects. If you need type-aware behavior, use the `@typescript-eslint` naming rule.

<Aside type="note" title="Design Philosophy">
	Naming rules shouldn't make your editor crawl. This one avoids type-checking, so it can run on every save.
</Aside>

## Configuration Workflow

<Steps>
1. **Identify Selectors**
   Choose which constructs to check (e.g., `interface`, `typeAlias`, `enum`).
2. **Select Formats** Define allowed casing formats (e.g., `PascalCase`, `camelCase`, `UPPER_CASE`).
3. **Define Custom Rules** Use regex to add specific constraints, like banning the `I` prefix on interfaces.
4. **Keep it cheap** This rule is meant to stay fast; save heavy checks for CI.
{/* oxfmt-ignore */}

</Steps>

## Selectors & Formats

| Selector    | Common Format | Description            |
| :---------- | :------------ | :--------------------- |
| `interface` | `PascalCase`  | Standard TS interfaces |
| `typeAlias` | `PascalCase`  | Type definitions       |
| `variable`  | `camelCase`   | Standard variables     |
| `constant`  | `UPPER_CASE`  | Global constants       |

## Why use this over TS-ESLint?

<CardGrid stagger>
	<Card title="No Type Graph" icon="bolt">
		Reads the AST only; it doesn't build the TypeScript program.
	</Card>
	<Card title="Regex Engine" icon="setting">
		Compiles regex up front and reuses it.
	</Card>
	<Card title="Memory Light" icon="rocket">
		Uses less memory than type-aware linting rules.
	</Card>
</CardGrid>
