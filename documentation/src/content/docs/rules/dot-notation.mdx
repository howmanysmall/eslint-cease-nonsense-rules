Prefer `object.property` over `object["property"]` when dot notation is legal.

## Rule Details :badge[TypeScript]{variant=note}

Most bracket access with a fixed string is harder to scan and easier to ignore in review. This rule pushes those cases
to dot notation, keeps the usual escape hatches from ESLint and TypeScript ESLint, and adds one `roblox-ts` option for
the cases where `obj["value"]` compiles but `obj.value` does not.

<Aside type="tip" title="Why the Roblox Escape Exists">
	In `roblox-ts`, bracket access to a `private` or `protected` member can compile cleanly in places where
	`object.member` would be rejected by TypeScript. This rule keeps reporting normal cases while skipping only those
	accesses that would become invalid after the autofix.
</Aside>

## Configuration

Use the default configuration if you just want standard dot notation checks. Add the `roblox-ts` options only if your
project relies on bracket access for members that TypeScript would reject with dot syntax.

| Option                                 | Type                        | Default      | Description                                                                                                 |
| -------------------------------------- | --------------------------- | ------------ | ----------------------------------------------------------------------------------------------------------- |
| `allowKeywords`                        | `boolean`                   | `true`       | Allow bracket notation for reserved keywords such as `obj["class"]`.                                        |
| `allowPattern`                         | `string`                    | `""`         | Allow bracket notation for property names that match a regular expression.                                  |
| `allowPrivateClassPropertyAccess`      | `boolean`                   | `false`      | Allow bracket notation for `private` class members.                                                         |
| `allowProtectedClassPropertyAccess`    | `boolean`                   | `false`      | Allow bracket notation for `protected` class members.                                                       |
| `allowIndexSignaturePropertyAccess`    | `boolean`                   | `false`      | Allow bracket notation for properties that are accessed through a string index signature.                   |
| `environment`                          | `"standard" \| "roblox-ts"` | `"standard"` | Switch the rule into `roblox-ts` mode so the compiler-aware escape can be enabled.                          |
| `allowInaccessibleClassPropertyAccess` | `boolean`                   | `false`      | Allow bracket notation when converting to dot notation would be rejected by TypeScript at that access site. |

## Why This Rule?

<CardGrid>
	<Card title="Readability" icon="book">
		`obj.property` is shorter, more familiar, and easier to scan than `obj["property"]` when both forms mean the
		same thing.
	</Card>
	<Card title="Compiler-Aware" icon="shield-check">
		In `roblox-ts`, the rule can leave bracket access alone when changing it would make the code fail type checking.
	</Card>
	<Card title="Auto-fixable" icon="wrench">
		Most violations can be cleaned up automatically with ESLint's `--fix` command.
	</Card>
</CardGrid>

## Visibility Escape

<Steps>
1. Resolve the member symbol from parser services and the object type.
2. Check upstream escape hatches first: private/protected/index-signature options.
3. In opt-in `roblox-ts` mode, ask the TypeScript checker whether dot access is legal at the current site.
4. Skip reporting only when dot access would be rejected there; otherwise report and autofix normally.

</Steps>

## Examples

<RuleExample type="fail" title="Incorrect">
	<Code
		code={[
			"interface PublicBox {",
			"\tname: number;",
			"}",
			"",
			"function incrementValue(object: PublicBox): void {",
			'\tobject["name"] += 1;',
			"}",
		].join("\n")}
		lang="ts"
	/>
</RuleExample>

<RuleExample type="pass" title="Correct">
	<Code
		code={[
			"interface PublicBox {",
			"\tname: number;",
			"}",
			"",
			"function incrementValue(object: PublicBox): void {",
			"\tobject.name += 1;",
			"}",
			"",
			"function incrementHidden(object: ExampleClass): void {",
			'\tobject["value"] += 1;',
			"}",
			"",
			"class ExampleClass {",
			"\tprivate value = 0;",
			"}",
		].join("\n")}
		lang="ts"
	/>
</RuleExample>
