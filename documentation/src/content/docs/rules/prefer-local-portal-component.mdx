Prefer a local `Portal` component over direct `createPortal(...)` calls.

## Rule details :badge[Suggestion]{variant=note}

This rule looks for `createPortal` calls from `@rbxts/react-roblox` or `react-dom` and reports them when the project
already has a matching local `Portal` component.

If the current file already imports that local component, the rule can replace the call with JSX. If the project has no
matching local component, or the rule cannot tell which one to use, it does nothing.

<Aside type="tip" title="When auto-fix runs">
	Auto-fix only runs in `.jsx` and `.tsx` files, and only when the current file already has exactly one matching local
	`Portal` import.
</Aside>

## Examples

<RuleExample type="fail" title="Incorrect">
		### Direct portal call

        	<Code
        		code={[
        			'import Portal from "../components/portal";',
        			'import { createPortal } from "@rbxts/react-roblox";',
        			"",
        			"export function Example(target: Instance) {",
        			"\treturn createPortal(<frame />, target);",
        			"}",
        		].join("\n")}
        		lang="tsx"
        	/>
    </RuleExample>

<RuleExample type="pass" title="Correct">
		### Local wrapper component

        	<Code
        		code={[
        			'import Portal from "../components/portal";',
        			"",
        			"export function Example(target: Instance) {",
        			"\treturn <Portal target={target}><frame /></Portal>;",
        			"}",
        		].join("\n")}
        		lang="tsx"
        	/>
    </RuleExample>

## What it checks

| Source                                                | Supported form                            |
| :---------------------------------------------------- | :---------------------------------------- |
| `@rbxts/react-roblox`                                 | `createPortal(children, target)`          |
| `react-dom`                                           | `createPortal(children, container)`       |
| `react-dom` or `@rbxts/react-roblox` namespace import | `ReactDOM.createPortal(children, target)` |
