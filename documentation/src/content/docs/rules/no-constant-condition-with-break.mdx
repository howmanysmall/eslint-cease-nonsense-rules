Disallow constant conditions, but allow constant true loops that have a real exit path.

## Rule details :badge[Problem]{variant=danger}

This rule reports constant conditions in `if` statements, ternaries, and loop tests.

It makes one exception for loops such as `while (true)`. Those are allowed when the loop can exit with `break`,
`return`, or a call you list in `loopExitCalls`.

Constant false loops are still reported. The loop never runs, so the exit analysis does not matter there.

<Aside type="tip" title="What counts as an exit">
	A `break` has to target the current loop. A `break` or `return` inside a nested function does not count.
</Aside>

## Options

`loopExitCalls` is an array of fully qualified call paths that should count as loop exits.

<Code
	code={[
		'import plugin from "eslint-plugin-cease-nonsense";',
		"",
		"export default [",
		"\t{",
		'\t\tplugins: { "cease-nonsense": plugin },',
		"\t\trules: {",
		'\t\t\t"cease-nonsense/no-constant-condition-with-break": [',
		'\t\t\t\t"error",',
		'\t\t\t\t{ loopExitCalls: ["coroutine.yield", "task.wait"] },',
		"\t\t\t],",
		"\t\t},",
		"\t},",
		"];",
	].join("\n")}
	lang="typescript"
	title="eslint.config.ts"
/>

## Examples

<RuleExample type="fail" title="Incorrect">
	<Code
		code={[
			"if (true) {",
			"\tdoSomething();",
			"}",
			"",
			"const value = true ? a : b;",
			"",
			"while (true) {",
			"\tdoSomething();",
			"}",
			"",
			"do {",
			"\tdoSomething();",
			"} while (false);",
		].join("\n")}
		lang="tsx"
	/>
</RuleExample>

<RuleExample type="pass" title="Correct">
	<Code
		code={[
			"while (running) {",
			"\tdoSomething();",
			"}",
			"",
			"while (true) {",
			"\tif (shouldStop) break;",
			"\tdoSomething();",
			"}",
			"",
			"function run() {",
			"\tfor (; true; ) {",
			"\t\tif (done) return;",
			"\t\twork();",
			"\t}",
			"}",
			"",
			"while (true) {",
			"\tcoroutine.yield();",
			"\twork();",
			"}",
		].join("\n")}
		lang="tsx"
	/>
</RuleExample>
