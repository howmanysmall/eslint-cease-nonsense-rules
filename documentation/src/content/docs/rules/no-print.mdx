Reports calls written as `print(...)`.

## Rule details :badge[Problem]{variant=danger}

This rule is simple on purpose. If the callee is an identifier named `print`, it reports the call. It does not report
`obj.print()` or `console.log(...)`, and it has no options.

<Aside type="tip" title="What to use instead">
	The rule message points you toward a logging package. Pick whatever logger your project already uses.
</Aside>

## Examples

<RuleExample type="fail" title="Incorrect">
		### Direct print call

        	<Code
        		code={[
        			'print("Request failed");',
        			'print("User logged in", userId);',
        		].join("\n")}
        		lang="ts"
        	/>
    </RuleExample>

<RuleExample type="pass" title="Correct">
	    	### Other logging calls are not checked

        	<Code
        		code={[
        			'logger.error("Request failed");',
        			'obj.print();',
        			'console.log("debug");',
        		].join("\n")}
        		lang="ts"
        	/>
    </RuleExample>
