Bans use of `print()` function calls. Use `log` instead for structured logging.

## Rule Details

The `print()` function is a low-level logging mechanism that lacks structure, context, and log level control. Using a proper logging system like [`Log`](https://sleitnick.github.io/RbxUtil/api/Log) provides better debugging capabilities and production logging.

<Aside type="tip" title="Why structured logging?">
	Structured logging helps you filter, search, and analyze logs in production. It provides timestamps, log levels, and
	context that `print()` simply cannot offer.
</Aside>

## Examples

### ❌ Incorrect

<Code
	code={[
		'print("Debug message");',
		'print("User logged in:", userId);',
		'print("Error:", error);',
		"",
		"function processData(data: unknown) {",
		'\tprint("Processing data:", data);',
		"\t// ...",
		"}",
	].join("\n")}
	lang="typescript"
	title="incorrect.ts"
/>

### ✅ Correct

<Code
	code={[
		'log.Info("Debug message");',
		'log.Info("User logged in:", userId);',
		'log.Error("Error:", error);',
		"",
		"function processData(data: unknown) {",
		'\tlog.Debug("Processing data:", data);',
		"\t// ...",
		"}",
	].join("\n")}
	lang="typescript"
	title="correct.ts"
/>

## Why Use Log Instead?

### Structured Logging

`Log` provides structured output with timestamps, log levels, and context:

<Code
	code={[
		"// print() - unstructured, no context",
		'print("Request failed");',
		"",
		"// log - structured with level and automatic timestamp",
		'log.Error("Request failed");',
		"// Output: [ERROR] 2025-12-31 12:34:56 - Request failed",
	].join("\n")}
	lang="typescript"
	title="structured-logging.ts"
/>

### Log Levels

Different log levels help filter messages in development vs production:

<Code
	code={[
		'log.Debug("Detailed debug info"); // Only in development',
		'log.Info("User action completed"); // General information',
		'log.Warning("Deprecated API used"); // Warnings',
		'log.Error("Operation failed"); // Errors',
	].join("\n")}
	lang="typescript"
	title="log-levels.ts"
/>

## When Not To Use It

<Aside type="caution">
	If you're writing quick scripts or prototypes where structured logging isn't necessary, you may disable this rule.
	However, it's recommended to use proper logging even in development.
</Aside>
