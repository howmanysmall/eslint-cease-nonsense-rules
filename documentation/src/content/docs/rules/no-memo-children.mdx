Reports `memo(...)` calls when the component props type includes a `children` property.

## Rule details :badge[Suggestion]{variant=note}

The rule checks imported `memo` calls from `react` or `@rbxts/react`, based on the configured `environment`. If the
memoized component's props type has a `children` property, it reports the call.

It resolves `children` through direct props, inherited interfaces, intersections, and unions. You can also allow
specific component names with `allowedComponents`.

<Aside type="tip" title="Known limitation">
	`PropsWithChildren<T>` imported from React is not always resolved by the current type lookup. An explicit props type or
	intersection is more reliable.
</Aside>

## Examples

<RuleExample type="fail" title="Incorrect">
		### Memoized component with children

        	<Code
        		code={[
        			'import { memo, ReactNode } from "@rbxts/react";',
        			"",
        			"interface CardProps {",
        			"\treadonly title: string;",
        			"\treadonly children?: ReactNode;",
        			"}",
        			"",
        			"const Card = memo<CardProps>((props) => {",
        			"\treturn <frame>{props.children}</frame>;",
        			"});",
        		].join("\n")}
        		lang="tsx"
        	/>
    </RuleExample>

<RuleExample type="pass" title="Correct">
	    	### Component without children

        	<Code
        		code={[
        			'import { memo } from "@rbxts/react";',
        			"",
        			"interface ButtonProps {",
        			"\treadonly label: string;",
        			"}",
        			"",
        			"const Button = memo<ButtonProps>((props) => {",
        			"\treturn <textbutton Text={props.label} />;",
        			"});",
        		].join("\n")}
        		lang="tsx"
        	/>
    </RuleExample>

## Options

| Option              | Type                        | Default       | Notes                                    |
| :------------------ | :-------------------------- | :------------ | :--------------------------------------- |
| `allowedComponents` | `string[]`                  | `[]`          | Component names the rule should skip.    |
| `environment`       | `"roblox-ts" \| "standard"` | `"roblox-ts"` | Picks `@rbxts/react` or `react` imports. |

```js
import ceaseNonsense from "eslint-plugin-cease-nonsense";

export default [
	{
		plugins: {
			"cease-nonsense": ceaseNonsense,
		},
		rules: {
			"cease-nonsense/no-memo-children": [
				"error",
				{
					allowedComponents: ["PortalHost"],
					environment: "standard",
				},
			],
		},
	},
];
```
