Reports import specifiers that are never used.

## Rule details :badge[Problem]{variant=danger}

This rule checks default imports, namespace imports, and named imports. When a binding is never referenced, it reports
it and can remove the unused specifier with `--fix`.

By default, references inside JSDoc also count as usage. Bare side-effect imports like `import "./init";` are ignored.

<Aside type="note" title="JSDoc references count by default">
	Tags like `@see`, `@link`, `@type`, `@param`, and `@returns` can keep an import alive when `checkJSDoc` is `true`.
</Aside>

## Options

| Option       | Type      | Default | What it does                               |
| ------------ | --------- | ------- | ------------------------------------------ |
| `checkJSDoc` | `boolean` | `true`  | Counts supported JSDoc references as usage |

## Examples

<RuleExample type="fail" title="Incorrect">
	<Code
		code={[
			'import { unusedFunction, usedFunction } from "./utils";',
			'import { AnotherUnused } from "./types";',
			"",
			"usedFunction();",
		].join("\n")}
		lang="tsx"
	/>
</RuleExample>

<RuleExample type="pass" title="Correct">
	<Code
		code={[
			'import { usedFunction } from "./utils";',
			'import { anotherUsedFunction } from "./more-utils";',
			"",
			"/**",
			" * @see {anotherUsedFunction}",
			" */",
			"usedFunction();",
		].join("\n")}
		lang="tsx"
	/>
</RuleExample>
