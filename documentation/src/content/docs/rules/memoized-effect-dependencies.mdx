Experimental rule that flags obviously unstable effect dependencies.

## Rule Details :badge[Experimental]{variant=caution}

This rule detects clearly un-memoized values inside effect dependency arrays. Unstable deps can cause unnecessary
rerenders or even loops when an effect updates something that gets recreated every render.

<Aside type="caution" title="Experimental">
	This rule is AST-only heuristics, not the React Compiler. Treat it as a hint and expect missed edge cases.
</Aside>

## Examples

<Tabs>
	<TabItem label="❌ Incorrect" icon="close">
		### Unmemoized dependencies
		<Code
			code={[
				'import { useEffect } from "@rbxts/react";',
				"",
				"function Component() {",
				"\tconst dep = {}; // new object every render",
				"\tuseEffect(() => {}, [dep]);",
				"}",
			].join("\n")}
			lang="tsx"
		/>
	</TabItem>
	<TabItem label="✅ Correct" icon="check">
		### Memoized dependencies
		<Code
			code={[
				'import { useEffect, useMemo } from "@rbxts/react";',
				"",
				"function Component() {",
				"\tconst dep = useMemo(() => ({}), []);",
				"\tuseEffect(() => {}, [dep]);",
				"}",
			].join("\n")}
			lang="tsx"
		/>
	</TabItem>
</Tabs>

### Modes

<Steps>
1. **definite** Only report obviously unstable dependencies (objects, arrays, functions, classes, `new`).
2. **moderate** Also report dependencies from unknown call expressions or non-`const` bindings.
3. **aggressive** Report any non-module, non-memoized binding.
