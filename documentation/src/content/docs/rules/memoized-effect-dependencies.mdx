Experimental rule that flags effect dependencies the rule can tell are not memoized.

## Rule details :badge[Experimental]{variant=caution}

This rule checks dependency arrays passed to `useEffect`, `useLayoutEffect`, and `useInsertionEffect`. By default it
uses the `roblox-ts` React environment and the `definite` mode.

In `definite` mode, it reports dependencies that are plainly unstable, such as inline objects, arrays, functions,
classes, and `new` expressions. The stricter modes widen the net:

<Steps>
1. **definite** Only report obviously unstable dependencies.
2. **moderate** Also report unknown call results and values declared with non-`const` bindings.
3. **aggressive** Report anything outside module scope unless the rule can tell it is stable.

{/* oxfmt-ignore */}

</Steps>

The rule treats values from `useMemo` and `useCallback` as memoized. It also knows that `useRef` and `useBinding` return
stable values, and that the second item from `useState`, `useReducer`, and `useTransition` is stable.

<Aside type="caution" title="Experimental">
	This is an AST-based heuristic, not the React Compiler. It can miss cases, and it can report code you want to keep.
</Aside>

## Examples

<RuleExample type="fail" title="Incorrect">
	### Unmemoized dependencies
	<Code
		code={[
			'import { useEffect } from "@rbxts/react";',
			"",
			"function Component() {",
			"\tconst dep = {}; // new object every render",
			"\tuseEffect(() => {}, [dep]);",
			"}",
		].join("\n")}
		lang="tsx"
	/>
</RuleExample>

<RuleExample type="pass" title="Correct">
	### Memoized dependencies
	<Code
		code={[
			'import { useEffect, useMemo } from "@rbxts/react";',
			"",
			"function Component() {",
			"\tconst dep = useMemo(() => ({}), []);",
			"\tuseEffect(() => {}, [dep]);",
			"}",
		].join("\n")}
		lang="tsx"
	/>
</RuleExample>

		"\t\t}",
		"\t]",
		"}",
	].join("\n")}
	lang="json"
	title="eslint.config.js"
/>

Add entries to `hooks` if you use custom effect-style hooks with a dependency array in a different argument position.
